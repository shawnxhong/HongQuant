     HongQuant 个人量化系统 —— 基建规划

     Context

     用户希望搭建个人中低频量化交易系统，覆盖 A 
     股/ETF、美股/ETF、加密货币三类市场，执行方式为半自动（系统产出信号 →
     人工下单），第一期即纳入 LLM 财报估值 / 交易建议 pipeline，已有 TradingView
      付费会员。

     当前仓库仅有规划类 Markdown（plans/quant_basics.md
     基金级综合稿、plans/TODO.md 10 个方向），无代码。目标是把 TODO.md
     落地为模块化基建，最大化复用成熟开源项目，避免重复造轮子。

     核心原则：
     1. 单人可维护：优先 Python 生态，组件数量收敛，能用 SaaS 就不自建。
     2. 信号先行，执行延后：半自动下单阶段不需要 OMS、FIX、低延迟。
     3. 三个市场统一抽象：数据/回测/信号层跨市场统一；只在数据源 adapter
     与执行通道处分叉。
     4. LLM 与量价并行双轨：量价 pipeline 走标准因子/回测流程；LLM 走"文档摄取 →
      结构化估值 → 交易建议"流程，二者在"仓位建议合成层"汇合。

     ---
     总体架构（建议分层）

     ┌─────────────────────────────────────────────────────────────┐
     │  D) 交付层   信号播报 / 日报 / 仓位建议 / TG/飞书/邮件         │
     ├─────────────────────────────────────────────────────────────┤
     │  C) 决策层   组合优化(Markowitz/风险平价) · 风控 · 仓位合成   │
     ├──────────────────────┬──────────────────────────────────────┤
     │  B1) 量价策略引擎    │  B2) LLM 研究 pipeline                │
     │    因子/信号/回测    │    财报/公告/新闻 → 估值&观点          │
     ├──────────────────────┴──────────────────────────────────────┤
     │  A) 数据层   行情(分钟/日) · 基本面 · 公告 · 宏观 · 另类       │
     ├─────────────────────────────────────────────────────────────┤
     │     基础设施   存储(Parquet+DuckDB/Postgres) · 调度 · 监控    │
     └─────────────────────────────────────────────────────────────┘

     ---
     各层选型（开源优先）

     基础设施

     - 语言/包管理：Python 3.11 + uv（或 poetry）；ruff + pytest。
     - 存储：
       - 行情历史 → Parquet 文件 + DuckDB 查询（零运维，个人规模最优解；比
     ClickHouse/TimescaleDB 更轻）。
       - 元数据/交易记录/回测结果 → SQLite 或 Postgres（Docker 单容器）。
       - 向量库（LLM 用）→ LanceDB 或 Chroma（本地文件型，零运维）。
     - 调度：Prefect 2 或 Dagster（推荐
     Prefect，代码侵入小，日级任务够用）；不建议 Airflow（过重）。
     - 监控/告警：轻量用 Loguru + Telegram Bot / 飞书 webhook；不推荐上
     Prom+Grafana，个人场景收益低。
     - 部署：本机 + 一台 VPS（阿里云/Vultr 2C4G 即可）跑定时任务；代码 Docker
     Compose 编排。

     A) 数据层

     - A 股 / ETF / 基本面：AkShare（免费广覆盖） + Tushare 
     Pro（付费但稳定，财报完整）+ baostock（备份）。
     - 美股：
       - 行情：yfinance（日线免费）/ Polygon.io（分钟级，$29/月起，可选）/
     Alpaca Data（账户免费）。
       - 基本面：SEC EDGAR（官方 XBRL，免费）+ Financial Modeling 
     Prep（FMP，$20/月可选）。
     - 加密货币：ccxt 统一拉取多交易所 K 线 + Binance/OKX REST 历史数据。
     - 公告/新闻（LLM 输入）：
       - A 股：巨潮资讯网（AkShare stock_zh_a_disclosure 等接口）。
       - 美股：SEC EDGAR 10-K/10-Q/8-K。
       - 加密：Twitter（X API v2 或 nitter 镜像）、交易所公告页。
     - TradingView 在此层的角色：TradingView 数据不开放 
     API，不适合作为数据源；但其Alerts → Webhook
     能作为"外部信号触发器"接入我方系统（见下）。

     B1) 量价策略引擎

     - 因子/研究：Qlib（微软开源，A 股生态最好，含因子库/回测/模型；可自定义数据
      handler 接入美股/加密） + pandas-ta / TA-Lib（指标计算）。
     - 回测：
       - 首选 vectorbt (pro 版可选，社区版够用) —— 向量化回测极快，适合多标的
     CTA/多因子。
       - 次选 Backtrader —— 事件驱动，贴近实盘语义，写策略直观。
       - 期货 CTA 深度场景可选 RQAlpha（Ricequant
     开源，国内期货友好），但一期可先不引入。
     - CTA 横截面动量 + 动量崩溃风控：在 vectorbt 上实现，参考 AQR
     论文；risk-off 信号用波动率/回撤阈值。
     - TradingView 复用点：
       - 用 Pine Script 做策略原型和可视化验证（画图调参比 Python 快）；
       - Pine 策略成型后翻译回 Python 纳入回测框架（保持研究闭环在本地）；
       - Alerts → Webhook：对于盘中触发型信号（趋势突破等），用 TV Alert 直推
     VPS 上的 FastAPI endpoint，作为半自动下单的"信号到达"入口。

     B2) LLM 研究 pipeline

     - 文档摄取：playwright / requests 拉取财报 PDF/HTML；pdfplumber +
     unstructured 解析。
     - 结构化：用 LangChain / LlamaIndex 编排；财报表格优先走 XBRL（SEC）或
     AkShare 已结构化字段，不要让 LLM 重新抽表格数字。
     - 估值 agent：DCF / 相对估值模板化（Python 函数）+ LLM 做"关键假设提取 +
     情景分析 + 质量打分"。
     - 模型：Claude Sonnet / DeepSeek V3（中文财报性价比高）/ 本地
     Qwen2.5-72B（隐私敏感场景）。
     - 记忆/检索：LanceDB 存公司×季度的摘要与事件流，供"交易建议
     agent"检索上下文。
     - 输出：每只关注标的每日/每周生成结构化
     JSON（估值区间、催化剂、风险、建议仓位方向），进入 C 层。

     C) 决策层

     - 组合优化：PyPortfolioOpt（Markowitz、Black-Litterman、HRP
     风险平价都有，API 成熟）。
     - 风控：自写轻量模块 ——
     单标的权重上限、行业/板块暴露、回撤熔断、相关性集中度。
     - 仓位合成：量价信号 + LLM 观点 → 加权（初期建议量价 70% / LLM
     30%，用实盘日志迭代）→ PyPortfolioOpt 求解目标权重 →
     与当前持仓比对生成调仓建议。

     D) 交付层

     - 每日任务：Prefect flow 定时（收盘后）跑：数据更新 → 信号计算 → LLM
     增量研究 → 组合优化 → 生成报告。
     - 推送：Telegram Bot / 飞书机器人 发送 Markdown
     日报（含当日信号、建议调仓、关键新闻）。
     - 仓位监控：手动维护一个持仓 YAML（或接券商账单 CSV
     导入），系统对比目标权重算出差异。
     - 可选 Web UI：Streamlit 做本地 dashboard（净值曲线、因子 IC、LLM
     观点时间线），避免自己写前端。

     Phase 0（1–2 周）脚手架

     Phase 1（2–4 周）量价闭环

     Phase 2（3–5 周）LLM 研究轨

     7. SEC EDGAR + AkShare 财报/公告摄取入库。

     agent，结果入 LanceDB。

     10. 接 TradingView Webhook（VPS 上起 FastAPI，收 Pine Alert → 入队 →
     与系统信号合并）。
     11. Streamlit 本地 dashboard。
     12. 若要升级到全自动：A 股用 QMT / 迅投 Ptrade，美股用 IBKR /

     TradingView 定位（最终建议）

     保留 TradingView 作为研究辅助与盘中信号触发器，不作为数据源或回测主战场：
     - ✅ 看盘 / 画线 / 指标原型：Pine Script 快速试想法。
     - ✅ Alert → Webhook 接入半自动执行：盘中突破类信号用 TV 触发，比自建 tick
     引擎简单。
     - ❌ 不用作主回测：Pine 回测有未来函数陷阱、多标的能力弱，统一到 vectorbt。
     - ❌ 不用作数据源：无官方 API，抓取违反 ToS。

     - 指标/因子：pandas-ta（纯 Python）为主；TA-Lib（C 加速）备选。
     - 因子分析：alphalens-reloaded 做 IC/IR、分层收益、换手率；quantstats 做

       a. ETF 横截面动量：SPDR 11 行业 ETF（XLK/XLF/...） + 商品

       b. 趋势 + 动量崩溃过滤：参考 AQR 2016 《Value and Momentum
     Everywhere》；用 12 月动量、跳过最近 1 月；叠加 6 月 Sharpe 或回撤阈值做

       c. 中低频均值回归（研究中）：ETF 对 pair trade 或 z-score 反转 ——

     B2) LLM 研究 pipeline

     流水线：公司关注池 → EDGAR 增量拉取 → 结构化 & RAG 索引 → Agent 输出 JSON →
      入 Postgres/LanceDB

     - 摄取：edgartools 拉 10-K/10-Q/8-K；unstructured + pdfplumber 备用 PDF。
     - 结构化：
       - 财务数字：直接从 XBRL companyfacts.json 读取（source of truth，不让 LLM
      抽）。
       - 段落语义：按 Item（1A 风险、7 MD&A、7A 市场风险等）切分，入 LanceDB。
     - Agent 层（LangChain 或 LlamaIndex workflow，任选其一，避免同时引入）：
       - 估值 Agent：DCF / 可比公司 / DDM 模板化为 Python 函数；LLM
     只负责假设提取（增长率、margin、re-investment
     rate）与情景分析（bull/base/bear）。
       - 质量 Agent：护城河、管理层、资本配置、会计质量评分（Piotroski
     F-score、Beneish M-score 作为量化 baseline + LLM 主观打分补充）。
       - 事件/催化剂 Agent：读 8-K、earnings call transcript（来源 Seeking
     Alpha/FMP transcript）→ 抽取近期催化剂、指引变化。
     - 模型选择：
       - 主力：Claude Sonnet 4.6（英文推理+结构化输出强）。
       - 性价比：DeepSeek V3 / Qwen-Max（中文 CLI 交互或批量任务）。
       - 本地隐私（可选）：Qwen2.5-72B / Llama-3.1-70B via vLLM。
     - 输出 schema（结构化 JSON，进入 C 层）：
     {
       "ticker": "NVDA", "as_of": "2026-04-12",
       "valuation": {"bull": 180, "base": 150, "bear": 95, "current": 140},
       "quality_score": 0.82,
       "thesis": "...",
       "catalysts": [...], "risks": [...],
       "suggested_tilt": "overweight|neutral|underweight",
       "confidence": 0.7
     }

     B3) 期权结构信号层（新增，一期核心）

     目标：把期权市场的定价/持仓信息转为正股/ETF 
     的方向、强度、风险特征，作为量价和 LLM 之外的第三路信号。

     每日 EOD 计算（per ticker）：
     - 定价结构
       - ATM IV（30D / 60D / 90D 插值）
       - IV Rank / IV Percentile（基于自建 1 年滚窗）
       - 期限结构斜率：front-month IV − 3M IV
       - Skew：25Δ Put IV − 25Δ Call IV（risk reversal）；wings 25Δ vs ATM
       - Variance risk premium：IV(30D) − 20D 已实现波动率
     - 流动 / 持仓
       - Put/Call Volume Ratio、Put/Call OI Ratio
       - 净 delta 流（用成交时的 bid/ask 方向近似）
       - Unusual Options Activity：volume > 3 × 10D avg 且 volume > OI 的
     contract
       - Max Pain（当周 / 当月）
     - Gamma Exposure (GEX)：
       - 用 OPRA OI + 做市商假设（market makers short puts, long calls →
     近似方向）计算每日 dealer gamma by strike。
       - 聚合得到 spot-GEX curve 与 zero-gamma flip level；公式参考
     SqueezeMetrics whitepaper。
     - 指数级信号（SPY/QQQ）
       - 0DTE 成交占比、VIX term structure（VIX/VIX3M）、skew index。

     实现：
     - 数据：IBKR API 每日收盘后 pull 关注池全链（chain + OI + IV +
     Greeks）；失败重试 + 增量落 Parquet。
     - 计算：纯 pandas + numpy 实现上述指标；BS/IV 用 py_vollib。无现成开源 GEX
     库，自写 ~200 行。
     - 输出：options_features(ticker, ts, feature_name, value) 宽表入 DuckDB，供
      B1/B2/C 层消费。
     - LLM 增强：options features 的极值（如 skew 骤升、GEX flip 跨越
     spot）自动入库为事件，LLM 交易建议 agent 在生成 thesis
     时检索这些事件并融入观点。

     C) 决策层

     - 组合优化：
       - 主用 PyPortfolioOpt（Markowitz / Black-Litterman / HRP），API 稳定。
       - 需要 CVaR / EVaR / Max Drawdown 目标时上 Riskfolio-Lib。
     - 风控（自写轻量模块 + 配置化）：
       - 权重上限：单标的 ≤ 10%，单行业 ≤ 30%，加密总敞口 ≤ 30%（可配置）。
       - 波动率目标：组合年化波动 15%（按 60 日滚窗调杠杆）。
       - 回撤熔断：组合回撤 > 10% → 减半仓位；> 15% → 清仓观望。
       - 相关性告警：关注池两两 60D 相关 > 0.85 触发集中度提醒。
     - 多源信号合成（量价 × LLM × 期权结构）：
       - 三路信号标准化到 z-score ∈ [-3, 3]，按权重合成（初期 量价 50% / LLM 25%
      / 期权 25%，用 out-of-sample 日志迭代权重）。
       - Black-Litterman 把 LLM "suggested_tilt" 作为 views 融入先验。
       - 输出：目标权重向量 → 与当前持仓对比 → 调仓单。

     D) 交付层

     - 每日调度（Prefect flow，美东时间）：
       - 16:30 股票 EOD 拉取 → 17:00 期权链快照 → 17:30 因子/信号 → 18:00 LLM
     增量研究 → 18:30 组合优化 → 19:00 Telegram 日报。
       - 加密每 4 小时增量 + 每日 00:00 UTC 对齐日线。
     - Telegram Bot：python-telegram-bot，Markdown
     日报含：当日信号表、调仓建议、关键事件（期权异动 /
     8-K）、持仓/净值图片（quantstats 生成）。
     - 本地 Dashboard：Streamlit（不引入前端）—— 净值曲线、因子 IC
     时间序列、期权 GEX 热力图、LLM 观点流。
     - 持仓对账：
       - IBKR 通过 ib_async 直接拉取 accountSummary 和 positions。
       - 富途/老虎：用户每周导出 CSV → portfolio/import.py 解析入库。
